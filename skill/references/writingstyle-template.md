# Writing Style Template

> The /team-setup skill generates a writingstyle.md for each user using this format. Sections are populated from the 7 questions (Step 5) and optional writing samples (Step 6).

---

## Output Format

Generate `writingstyle.md` in this structure:

```markdown
# Writing Style - {{NAME}}

> Generated by /team-setup on {{DATE}}. Edit anytime to refine. Add writing samples to improve accuracy.

## Voice Summary

{{2-3 sentence summary of their overall writing voice. Example: "Direct and concise. Opens casual, closes fast. Uses humor with people he knows but keeps external comms professional. Bottom-line-first structure."}}

## Patterns

**Greetings:** {{How they open emails/messages. Example: "First name only for colleagues ('Jordan,'), 'Hi [name],' for external."}}

**Sign-offs:** {{How they close. Example: "Just their name. No 'Best' or 'Thanks' prefix."}}

**Length:** {{Typical message length. Example: "Short. 2-3 sentences for most emails. Longer for project updates but still tight."}}

**Structure:** {{Bullets vs prose, how they organize. Example: "Prose for most things. Bullets only for lists of 3+ items."}}

**Delivering bad news:** {{How they handle difficult messages. Example: "Direct. States the issue, then the plan. Doesn't over-apologize."}}

**Humor:** {{When and how they use humor. Example: "Sometimes with people he knows. Dry, never forced."}}

**Energy signals:** {{How excitement or urgency shows up. Example: "Short punchy sentences. Might capitalize a word for emphasis. Adds exclamation marks sparingly."}}

## Sample Patterns (if samples provided)

{{Only include this section if the user provided writing samples in Step 6.}}

**Sentence length:** {{Average word count per sentence. Example: "10-14 words. Rarely exceeds 20."}}

**Vocabulary:** {{Casual vs formal, jargon patterns. Example: "Casual-professional. Uses industry terms without explanation internally."}}

**Punctuation habits:** {{Notable patterns. Example: "Dashes instead of semicolons. No em dashes. Rare exclamation marks."}}

**Argument structure:** {{How they build a case. Example: "Bottom-line first, then supporting points. Never buries the lead."}}

**Unique markers:** {{Anything distinctive. Example: "Uses '+' instead of 'and' in quick notes. Capitalizes for emphasis."}}

## Rules for Claude

When writing as {{NAME}}:
{{3-5 specific rules derived from the profile. Examples:}}
- Open emails with first name only to colleagues, "Hi [name]," to external contacts
- Keep emails to 2-3 sentences unless the topic requires more
- Close with just "{{NAME}}." - no prefix
- Use contractions always
- Match their energy level to the context - casual in Slack, precise in formal docs
```

---

## Generation Logic

### From Questions Only (No Samples)

Map each answer directly:

| Question | Maps to |
|----------|---------|
| Q1: Email opening | Greetings |
| Q2: Sign-off | Sign-offs |
| Q3: Email length | Length |
| Q4: Bullets or prose | Structure |
| Q5: Bad news delivery | Delivering bad news |
| Q6: Humor usage | Humor |
| Q7: Excitement expression | Energy signals |

Write the Voice Summary by combining the strongest signals from all answers.

Generate 3-5 Rules for Claude based on the most distinctive patterns.

### From Questions + Samples

1. Analyze samples first for concrete patterns (sentence length, vocabulary, punctuation, structure)
2. Compare to self-reported answers from questions
3. **Samples win when they conflict** - people are bad at describing their own voice
4. Populate the "Sample Patterns" section
5. Merge both sources into the Voice Summary and Rules

### Review Step

After generating, show the full profile and ask: "Here's your writing style profile. Read it and tell me what's off - I'll adjust."

Iterate until they're satisfied. Common edits:
- "I'm more formal than that with external people" -> add context-switching rule
- "I don't really use humor" -> tone down the humor line
- "That sign-off is wrong" -> fix it

Save final version to `~/.claude/memory/writingstyle.md`.
